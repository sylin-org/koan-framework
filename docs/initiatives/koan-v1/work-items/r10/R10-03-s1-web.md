# R10-03 — Graduate S1.Web

- Tranche: `T7B — maintained-sample graduation`
- Status: `passed`
- Depends on: R10-01 standard and R10-02 inventory
- Owner: minimal Entity web/relationship/cache curriculum and executable proof

## Business sentence

“Create tasks owned by people and categories, break them into actionable items, and retrieve the same
relationship context from one task, a finite set, or a bounded async stream.”

## Smallest honest host

```csharp
var builder = WebApplication.CreateBuilder(args);
builder.Services.AddKoan();
var app = builder.Build();
await app.RunAsync();
```

S1 earns no application module: it has no composition-time business policy or startup work. Its deliberate
concept budget is Entity models, `[Parent]`, generated controllers, scalar/set/stream `Relatives()`, and one
`[Cacheable]` opt-in that demonstrates reference-plus-decoration composition.

## Exploration evidence

### Current intent and references

- Entities: `User`, `Category`, `Todo`, `TodoItem`; Todo has two parents and TodoItem is its child.
- Controllers: four generated Entity HTTP surfaces plus custom seeding, relationship, stream, and cleanup routes.
- Direct capabilities: SQLite, JSON, Web, Web.Extensions, Admin, and Cache, plus an unused ASP.NET SPA package.
- Explicit `[DataAdapter("sqlite")]` on every Entity resolves the ambiguity created by referencing two data adapters.
- Static AngularJS UI, launch profile, four local/container start scripts, Dockerfile/compose, cleanup script, and
  `requests.http` form a large sample-tooling surface.

### Baseline result

- Strict Release build passes with 0 warnings/errors.
- A real in-memory SQLite host serves the dashboard and facts without collection failure.
- The required three-step random seed path and both relationship-demo endpoints return 200.
- `Program.cs` still contains sample logging/lifecycle decoration, exception/rate-limit policy, manual
  `AppHost.Current`, data-directory creation, commented alternate setup, and browser-launch ceremony.
- README teaches helpers and “console bootstrap” rather than the application, uses an incorrect repository path,
  and describes stale generic `Relatives<T,TKey>()` UI snippets.
- `TodoCsvTransformer.cs` is an intentionally empty placeholder.

## Coalescence decision

Rebuild S1 around one meaningful relationship story:

- keep SQLite as the sole data intent; remove JSON and all per-Entity adapter attributes;
- keep Web.Extensions and Cache; remove direct/transitive ceremony, Admin, and the unused SPA package;
- make `Program.cs` the four-line host with no app-specific middleware policy;
- retain four business-readable Entities and `[Cacheable]` only on Category;
- reduce standard controllers to declarations;
- give `TodoController` one deterministic `reset-demo` workflow, one scalar relationship-context route, and one
  finite-set and bounded relationship-stream routes;
- use consistent plural HTTP routes;
- rebuild the static dashboard around the same reset/context story and current non-generic `Relatives()` grammar;
- delete the placeholder, redundant launch/container/cleanup scripts, Docker claim, and old scaffold;
- retain one corrected standard .NET launch profile and one `requests.http` contract.

No repository, service layer, custom provider selector, module, route-constant hierarchy, or generic sample harness
is earned. The relationship compiler/runtime remains Data-owned; S1 merely states the business graph.

## Focused acceptance

1. One standard `dotnet run` from the sample directory reaches the dashboard and meaningful relationship result.
2. The project references only deliberate capabilities; SQLite is elected from reference intent with no Entity
   adapter decorations.
3. Application source is the four-line host, four Entities, four controllers, and one deterministic demo workflow.
4. Scalar, finite-set, and bounded-stream relationship results agree on parent/child meaning and preserve source order.
5. Category cache activation and SQLite/relationship decisions are inspectable in startup/facts without collection
   failure.
6. A sample-local cumulative test proves reset → scalar context → stream context → generated HTTP/facts.
7. Strict build, README/dashboard/requests, solution/test membership, docs, diff, and privacy gates pass.

## Unsupported/non-goals

- S1 does not teach auth, admin, rate limiting, custom transformers, external cache, provider switching, or
  containers; other samples own those concepts.
- The local memory cache floor does not imply distributed cache/coherence behavior.
- Relationship streaming is provider-bounded and consumer-paced; S1 does not promise arbitrary adapter parity or
  cross-provider joins.

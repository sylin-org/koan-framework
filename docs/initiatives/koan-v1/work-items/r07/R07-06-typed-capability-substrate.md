# R07-06 — Typed capability substrate

- Tranche: `T6 — semantic capability ring`
- Status: `passed`
- Depends on: R07-05
- Unlocks: faithful local Entity Transport
- Owner: Data.Core cardinality normalization and pillar-owned execution

## Meaningful outcome

Koan has one smaller foundation for future pointwise Entity capabilities:

```csharp
EntityCardinality.One(order);
EntityCardinality.Many(orders);
EntityCardinality.Stream(Order.QueryStream(order => order.Ready));
```

Each form remains a lazy `IAsyncEnumerable<TEntity>` preserving order and multiplicity. The helper
does not execute callbacks, choose providers, carry feature bags, batch work, create receipts, or
define failure policy. Events, Transport, AI, Cache, and later admitted pillars can normalize their
source through this seam while retaining their own terminal semantics and outcomes.

The generic public Pipeline DSL is deleted without an alias. It mixed Data persistence, AI calls,
vector writes, Messaging, and observability through a mutable string-keyed envelope while providing
no stable aggregate contract. Canon and Media retain their domain-specific pipelines because those
types describe real, bounded domain processes rather than generic Entity composition.

## Architecture

- `IEntity` is a key-agnostic semantic marker; `IEntity<TKey>` retains the identifier contract and
  derives from it. The marker is a type constraint only, never a runtime discovery mechanism.
- `EntityCardinality` owns scalar, finite-set, and async-stream normalization. All three return the
  standard async-stream substrate; finite sources remain lazy and enumerate once.
- Cancellation, early disposal, ordering, multiplicity, and null-item diagnostics are part of the
  normalization contract.
- Data.Core owns no cross-pillar execution. There is no callback registry, envelope, feature bag,
  branching DSL, provider election, retry, atomicity, or receipt in the substrate.
- Ordinary `[Embedding]` writes remain Entity Lifecycle behavior. Explicit finite-set rebuilds use
  `EmbeddingMigrator.ReEmbed`; whole-collection model transitions use `ReEmbedAll<T>`.
- Migration performs vector-only writes so it does not re-save the Entity and recursively fire
  persistence Lifecycle. `MigrationResult` reports partial outcomes without promising retry,
  collection atomicity, or rollback.
- S0 expresses its mutation as the business-named `CompleteTaskTitles` operation. S5 imports only
  business data; `[Embedding(Async = true)]` owns ordinary indexing, while its explicit vector-only
  operation delegates to the migrator.

## Principal deletions

- `Koan.Core.Pipelines`: builder, envelope, branch builders, feature keys, and fluent extensions.
- Generic Pipeline extensions in Data.Core, AI, Data.Vector, Messaging.Core, and Core observability.
- Public Flow/Semantic Pipeline guidance that taught the removed composition model.
- S5's duplicate manual embedding text, cache, vector metadata, and generic pipeline orchestration.

## Acceptance

- The substrate normalizes only Entity cardinality and cannot absorb pillar policy accidentally.
- No general public Pipeline/container type remains.
- The two real application consumers become more business-readable and retain truthful behavior.
- Ordinary embedding and explicit rebuilds have different, named owners and observable outcomes.
- Current public guidance teaches Lifecycle, migrators, Jobs, or business-named loops rather than the
  removed DSL; historical ADRs are clearly amended.
- Intentional package compatibility tiers and composition lockfiles are automation-owned.
- Focused semantic, consumer, package-lineage, docs, diff, and privacy gates pass. The public-release
  certification ratchet is not rerun for this ordinary child.

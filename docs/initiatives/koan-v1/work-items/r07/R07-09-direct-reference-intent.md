# R07-09 — Direct reference intent

- Tranche: `T6 — semantic capability ring`
- Status: `passed`
- Depends on: R07-08
- Unlocks: truthful Communication connector eligibility and election
- Owner: Core build provenance and composition inspection

## Meaningful outcome

Koan can distinguish what the application directly chose from what its dependency graph merely
contains. Every supported executable build embeds an immutable direct-reference manifest and records
the same facts in `koan.lock.json`. A future connector can therefore become an automatic candidate
only when the application directly references it; a transitive dependency cannot silently change
network reach.

The application writes no registration or routing code. Package and source-checkout builds both
participate through the existing Core build target.

## Architecture

- Core snapshots application-authored `PackageReference` and `ProjectReference` items during MSBuild
  evaluation, before project resolution can add transitive items.
- `koan.modules.manifest` continues to record the resolved Koan module closure for loading.
  `koan.references.manifest` separately records direct intent as `package|id` or `project|id`.
- `KoanApplicationReferenceManifest` is an immutable host service available before pillar
  initializers. `IsPresent=false` means provenance is unknown, never an asserted empty reference set.
- A present malformed resource fails with corrective rebuild guidance. Missing provenance remains
  compatible but cannot authorize automatic external-provider selection.
- Composition lock schema 2 records `directReferences` in both the checked-in file and resolved twin.
  Module presence and reference intent remain visibly distinct.
- This slice adds no Communication adapter SPI, channel API, broker, fallback, or election. Those
  abstractions must be earned by a real connector conformance proof.

## Explicit non-claims

Direct reference intent is an election input, not an election result. Koan does not yet claim external
Communication providers, multi-connector selection, logical channels, distributed reach, retries,
durability, broker settlement, or RabbitMQ parity.

# Migration Guide: Koan Framework v0.2.x → v0.6.x

This guide covers the breaking API changes introduced between **v0.2.18** (last stable pre-0.6 release)
and **v0.6.3**. The principles have not changed — Entity-First, Reference = Intent, Provider Transparency
— but several concrete APIs were renamed or replaced.

---

## Quick Summary

| Area | Change |
|------|--------|
| AI facade | `IAi` + `ChatAsync()` replaced by static `Client` + `PromptAsync()` / `Client.ChatAsync()` |
| Boot reporting | `BootReport` replaced by `ProvenanceModuleWriter` |
| AI adapter contract | Monolithic `IAiAdapter` split into `IChatAdapter`, `IEmbedAdapter`, `IOcrAdapter` |
| AI namespaces | Single `Koan.AI` expanded to 9 domain modules |
| Data.AI | New package — embeddings lifecycle + media analysis |
| .NET target | `net9.0` → `net10.0` |

---

## Breaking Changes

### 1. AI Facade: `IAi` → static `Client`

**v0.2.x (removed):**

```csharp
// Injected via DI — no longer works
public class MyService(IAi ai)
{
    async Task DoWork(CancellationToken ct)
    {
        var response = await ai.ChatAsync(new AiChatRequest
        {
            Model = "gpt-4",
            Messages = ...,
            SystemPrompt = "..."
        }, ct);
    }
}
```

**v0.6.x (correct):**

```csharp
// No injection needed — use the static Client facade
public class MyService
{
    async Task DoWork(CancellationToken ct)
    {
        // Simple message
        var response = await Client.ChatAsync("your message", ct);

        // With system prompt — use the conversation builder
        var response = await Client.Converse()
            .WithSystem("You are a helpful assistant.")
            .WithUser("your message")
            .PromptAsync(ct);

        // Streaming
        await foreach (var chunk in Client.StreamAsync("your message", ct))
            Console.Write(chunk.Content);
    }
}
```

**If you need a testable seam**, inject `IAiPipeline` and call `PromptAsync` / `StreamAsync`:

```csharp
public class MyService(IAiPipeline ai)
{
    public Task<AiChatResponse> Ask(string message, CancellationToken ct) =>
        ai.PromptAsync(new AiChatRequest
        {
            Messages = [new AiMessage { Role = "user", Content = message }]
        }, ct);
}
```

> `IAi` is **removed**. `ChatAsync` is **not a member of any injectable type**. `PromptAsync` is the
> correct method on `IAiPipeline`.

### 2. Boot Reporting: `BootReport` → `ProvenanceModuleWriter`

**v0.2.x (removed):**

```csharp
public void Describe(BootReport report, IConfiguration cfg, IHostEnvironment env)
{
    report.AddModule(ModuleName, ModuleVersion);
    report.AddNote("Services registered");
    report.AddWarning("Optional feature missing");
}
```

**v0.6.x (correct):**

```csharp
public void Describe(ProvenanceModuleWriter module, IConfiguration cfg, IHostEnvironment env)
{
    module.Describe(ModuleVersion, "My module description");
    module.AddNote("Services registered");                          // extension method
    module.SetStatus("degraded", "Optional feature not configured"); // replaces AddWarning
}
```

**Full `ProvenanceModuleWriter` API:**

| Method | Replaces |
|--------|---------|
| `module.Describe(version, description)` | `report.AddModule(name, version)` |
| `module.AddNote(message)` | `report.AddNote(message)` |
| `module.SetStatus("degraded", detail)` | `report.AddWarning(message)` |
| `module.SetSetting(key, builder)` | _(new — structured settings)_ |
| `module.SetTool(name, builder)` | _(new — register endpoints)_ |

> `BootReport` is **removed**. `AddModule`, `AddWarning` do not exist on any type.

### 3. AI Adapter Contract: `IAiAdapter` → Split Interfaces

The monolithic `IAiAdapter` has been split into focused contracts.

**v0.2.x:**

```csharp
public class MyAdapter : IAiAdapter
{
    public Task<AiChatResponse> ChatAsync(AiChatRequest request, CancellationToken ct) { ... }
    public Task<float[]> EmbedAsync(string text, CancellationToken ct) { ... }
    public Task<string> OcrAsync(Stream image, CancellationToken ct) { ... }
}
```

**v0.6.x — implement only the capabilities your adapter provides:**

```csharp
// Chat-only adapter
public class MyChatAdapter : IChatAdapter
{
    public Task<AiChatResponse> ChatAsync(AiChatRequest request, CancellationToken ct) { ... }
    public IAsyncEnumerable<AiChatChunk> StreamAsync(AiChatRequest request, CancellationToken ct) { ... }
}

// Embed-only adapter
public class MyEmbedAdapter : IEmbedAdapter
{
    public Task<float[]> EmbedAsync(string text, CancellationToken ct) { ... }
}

// OCR-only adapter
public class MyOcrAdapter : IOcrAdapter
{
    public Task<string> OcrAsync(Stream image, CancellationToken ct) { ... }
}
```

> Adapters that implement all three can implement all three interfaces. The resolver selects the
> correct adapter per category automatically.

### 4. .NET Target: `net9.0` → `net10.0`

All Koan packages target `net10.0` from v0.6.0 onward. Update your `csproj`:

```xml
<!-- Before -->
<TargetFramework>net9.0</TargetFramework>

<!-- After -->
<TargetFramework>net10.0</TargetFramework>
```

## New Modules (no migration needed — additive)

These packages are new in v0.6.x and have no v0.2.x equivalents. Add them incrementally:

| Package | Purpose |
|---------|---------|
| `Sylin.Koan.Data.AI` | Embeddings lifecycle, `[Embedding]`, `[MediaAnalysis]`, `MediaAnalysisMetadata` |
| `Sylin.Koan.AI.Agents` | Autonomous agents, entity tool auto-generation, ReAct reasoning |
| `Sylin.Koan.AI.Compute` | GPU/compute discovery, workload placement |
| `Sylin.Koan.AI.Eval` | Model evaluation, quality gates, drift detection |
| `Sylin.Koan.AI.Models` | Model catalog, pull/convert/deploy/version lifecycle |
| `Sylin.Koan.AI.Orchestration` | RAG chains, branching, parallel execution |
| `Sylin.Koan.AI.Prompt` | Versioned prompt storage, variable interpolation, A/B testing |
| `Sylin.Koan.AI.Review` | Human-in-the-loop review queues |
| `Sylin.Koan.AI.Training` | LoRA fine-tuning, DPO/RLHF alignment, dataset management |

## New Configuration Keys (AI)

AI configuration keys changed to support per-category routing:

| v0.2.x | v0.6.x |
|--------|--------|
| `Koan:AI:Providers:Primary:Type` | `Koan:Ai:Chat:Source` |
| `Koan:AI:Providers:Primary:Model` | `Koan:Ai:Chat:Model` |
| `Koan:AI:Providers:Fallback:Type` | `Koan:Ai:Embed:Source` (separate from Chat) |

Each category (Chat, Embed, Ocr) can now route to a different source independently.

---

## Non-Breaking Changes (informational)

These changed between v0.2.x and v0.6.x but are backward-compatible or additive:

- `Configuration.ReadWithSource<T>()` — new method returning `ConfigurationValue<T>` with source attribution; plain `Read<T>()` still works unchanged
- `IBatchSet<TEntity, TKey>` — new explicit type for `Entity.Batch()` return; fluent API unchanged
- Transfer enums (`TransferKind`, `DeleteStrategy`, `MirrorMode`) — documentation added, no value changes

## Migration Checklist

- [ ] Update all `<TargetFramework>` to `net10.0`
- [ ] Update all Koan package versions to `0.6.3`
- [ ] Replace `IAi` DI injection with static `Client` or `IAiPipeline`
- [ ] Replace `ai.ChatAsync(new AiChatRequest {...})` with `Client.ChatAsync(message, ct)` or `Client.Converse()...PromptAsync(ct)`
- [ ] Replace `BootReport report` parameter with `ProvenanceModuleWriter module`
- [ ] Replace `report.AddModule()` with `module.Describe(version, description)`
- [ ] Replace `report.AddWarning()` with `module.SetStatus("degraded", detail)`
- [ ] If you implemented `IAiAdapter`, split into `IChatAdapter` / `IEmbedAdapter` / `IOcrAdapter` as appropriate
- [ ] Update AI configuration keys from `Koan:AI:Providers:*` to `Koan:Ai:{Category}:*`

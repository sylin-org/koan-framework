# Migration Guide: Koan.Media v0.8 → v0.9 (MEDIA-0004)

Koan.Media v0.9 is a **break-and-rebuild release** that replaces
the DX-0047 operator/transform/cache surface with the MEDIA-0004
recipe pipeline. This guide covers every removed type and its
replacement.

The rebuild was driven by a single defect: every transform in v0.8
silently re-encoded its output as JPEG, destroying animation,
alpha, and color depth in the process. The v0.9 pipeline preserves
source format by default — animated WebP / GIF / APNG round-trip
through resize and crop with frames intact; transparent PNG keeps
its alpha; wide-gamut sources stay in their color space.

For the design rationale, see [MEDIA-0004](../decisions/MEDIA-0004-recipe-pipeline.md).
For the walkthrough, see [Media Recipes How-To](media-recipes-howto.md).
For the reference surface, see [Media Pillar Reference](../reference/media/index.md).

---

## Quick Summary

| Area | v0.8 | v0.9 |
|---|---|---|
| Pipeline entry | `stream.ResizeFit(...).ConvertFormat(...).ToBytes()` | `stream.AsMedia().ResizeFit(...).ToBytesAsync()` |
| Single-output | `Task<Stream>` per step | `Task<MediaOutput>` once at the terminus |
| Multi-output | `Result().Branch()` (N decodes) | `MaterializeAsync(...)` (one decode, N encodes) |
| Format policy | Every transform → JPEG | Source format preserved unless `FlattenTo` |
| HTTP routes | Per-entity `MediaContentController<T>` | Single recipe-based `MediaController` + per-entity `StorageMediaController<T>` |
| Variant transforms | Operator registry + ad-hoc query params | Recipe registry + URL grammar with allowlist |
| Variant addressing | `/api/media/{id}?w=300&fit=cover` | `/media/{id}/{recipe}?w=300` or `/media/{id}?w=300&fit=cover` |
| Recipe declaration | Implicit, derived from query params | `[MediaRecipe]` attribute + `Koan:Media:Recipes` config |
| Cache backend | `IMediaTransformCache` + `MediaTransformCacheOptions` | Variant key = `(sourceHash, recipeFingerprint)`; CDN / browser cache via standard headers |
| Removed types | `MediaTask*`, `MediaTransformSpec`, `MediaVariant`, `IMediaOperator*`, `MediaOperatorRegistry`, `MediaTransformCache`, `MediaContentController`, `TransformResult`, `StreamTransformExtensions`, `MediaContentOptions`, `MediaTransformOptions`, `SignatureUtility` | — |

## 1. Replace `StreamTransformExtensions` with `stream.AsMedia()`

The single biggest change. The v0.8 fluent API hard-coded JPEG on
every method. The v0.9 pipeline is lazy until materialised and
preserves format by default.

### Before (v0.8)

```csharp
using Koan.Media.Core.Extensions;

await using var source = File.OpenRead("photo.webp");
await using var bounded = await source.ResizeFit(800, 600, ct);
await using var encoded = await bounded.ConvertFormat("jpeg", 85, ct);
var bytes = await encoded.ToBytes(ct);
```

### After (v0.9)

```csharp
using Koan.Media.Core.Pipeline;

await using var source = File.OpenRead("photo.webp");
var output = await source.AsMedia()
    .ResizeFit(800, 600)
    .ToBytesAsync(ct);

// output.Bytes — the encoded bytes
// output.ContentType — "image/webp" because the source was animated WebP
// output.Format — "webp"
// output.FrameCount — original frame count, preserved
```

**Key differences:**

- `AsMedia()` is the single entry point. Subsequent verbs chain on
  `IMediaPipeline`, not on `Stream`.
- The pipeline owns the source stream and disposes it at the first
  terminal call.
- The terminal `ToBytesAsync()` returns `MediaOutput` (bytes +
  metadata), not a raw stream.
- If you genuinely want JPEG out, call `.FlattenTo("jpeg", 85)`
  instead of `.ConvertFormat("jpeg", 85)`. The `FlattenTo` name
  signals that animation and alpha may be dropped; the engine logs
  at Information when it runs.

### Per-operation mapping

| v0.8 method | v0.9 replacement | Notes |
|---|---|---|
| `Resize(int w, int h)` | `Resize(int? w, int? h, double dpr)` | Same semantics. Single-axis preserves aspect. |
| `Resize(double sx, double sy)` | `Resize(int? w, int? h, dpr: scale)` | Or compute target dimensions yourself. |
| `ResizeFit(int maxW, int maxH)` | `ResizeFit(int maxW, int maxH)` | Same. |
| `ResizeCover(int w, int h, CropFrom)` | `ResizeCover(int w, int h, Position?)` | `CropFrom` enum replaced by `Position` value-type. |
| `ResizeX(int w)` | `Resize(width: w)` | Single-axis form. |
| `ResizeY(int h)` | `Resize(height: h)` | Single-axis form. |
| `Crop(int x, int y, int w, int h)` | `Crop(CropSpec.PixelsAt(w, h, x, y))` | Or `?crop=WxH+X,Y` via URL. |
| `Crop(Rectangle)` | `Crop(CropSpec.PixelsAt(...))` | `Rectangle` is still available under `Koan.Media.Core.Model`. |
| `CropCenter(int w, int h)` | `Crop(CropSpec.Pixels(w, h))` (defaults to center) | |
| `Crop(double aspect, CropFrom)` | `Crop("16:9")` or `Crop(CropSpec.Aspect(16, 9))` | |
| `CropSquare(CropFrom)` | `Crop("square")` or `Crop(CropSpec.Square)` | |
| `Pad(int w, int h, Color?)` | `Shape(crop: ..., fit: Fit.Contain, background: ...)` | The shape stage unified pad + fit. |
| `Pad(PadTo, Color?)` | `Shape(crop: aspect, fit: Fit.Contain, background: ...)` | |
| `AutoOrient()` | `AutoOrient()` | Runs by default if not declared; pass `keep: true` to skip. |
| `Rotate(int deg)` | `Rotate(int deg)` | Same. |
| `FlipHorizontal()` / `FlipVertical()` | `FlipHorizontal()` / `FlipVertical()` | Same. |
| `ConvertFormat(string fmt, int q)` | `EncodeAs(string fmt, int q)` | **Non-destructive when target supports source's properties** — replaces the v0.8 always-JPEG behaviour. For explicit destruction, use `FlattenTo`. |
| `OptimizeQuality(int q)` | `PreserveFormat(quality: q)` | Same-format re-encode at the given quality. |

### Terminal operations

| v0.8 terminal | v0.9 replacement |
|---|---|
| `ToBytes()` | `ToBytesAsync()` returns `MediaOutput` — use `.Bytes` for the raw array |
| `ToBase64()` | `Convert.ToBase64String((await pipeline.ToBytesAsync()).Bytes)` |
| `Result()` + `.Branch()` | `MaterializeAsync(bundle => ...)` — see §2 |
| `StoreAs<TEntity>()` | Call `ToBytesAsync()` then `TEntity.Store(output.Bytes, output.ContentType, ...)` |

## 2. Replace `Result().Branch()` with `MaterializeAsync`

The v0.8 branching pattern materialised an intermediate
`TransformResult` and cloned its bytes per branch. Every branch
then re-decoded its own copy, paying the decode cost N times.
`MaterializeAsync` decodes the source once and clones the in-memory
image per branch — 5-10× faster on typical multi-variant
generation.

### Before (v0.8)

```csharp
using var result = await source
    .OpenRead()
    .AutoOrient()
    .OptimizeQuality(85)
    .Result();

var small  = await result.Branch().ResizeX(150).StoreAs<ThumbnailSmall>();
var medium = await result.Branch().ResizeX(300).StoreAs<ThumbnailMedium>();
var large  = await result.Branch().ResizeX(600).StoreAs<ThumbnailLarge>();
```

### After (v0.9)

```csharp
await using var source = await photo.OpenReadAsync();
var bundle = await source.AsMedia()
    .AutoOrient()
    .MaterializeAsync(b => b
        .Add("small",  v => v.Resize(width: 150).PreserveFormat(85))
        .Add("medium", v => v.Resize(width: 300).PreserveFormat(85))
        .Add("large",  v => v.Resize(width: 600).PreserveFormat(85)));

// bundle.Variants["small"].Bytes — encoded small variant
// bundle.Variants["small"].ContentType — content type for storage
foreach (var (name, output) in bundle.Variants)
{
    var entity = name switch
    {
        "small"  => await ThumbnailSmall.Store(output.Bytes, output.ContentType, ...),
        "medium" => await ThumbnailMedium.Store(output.Bytes, output.ContentType, ...),
        "large"  => await ThumbnailLarge.Store(output.Bytes, output.ContentType, ...),
        _ => null
    };
}
```

The `AutoOrient()` step lifts to the parent pipeline so every
branch starts from the oriented image. Steps inside the branch
configure that branch's pipeline only.

## 3. Replace `MediaContentController<TEntity>` Inheritance

v0.8's `MediaContentController<TEntity>` bundled raw-bytes streaming
AND on-the-fly transforms in one class. v0.9 splits the
responsibilities:

- **Raw byte streaming per entity** → inherit `StorageMediaController<TEntity>`.
  Provides HEAD / GET / Range / If-None-Match / If-Modified-Since /
  Content-Disposition / Cache-Control.
- **On-the-fly transforms** → use the global `MediaController` at
  `/media/{id}` (recipe pipeline). One controller per app, not one
  per entity.

### Before (v0.8)

```csharp
[Route("api/previews")]
public sealed class PreviewsController : MediaContentController<PreviewImage>
{
    public PreviewsController(IOptions<MediaContentOptions> options) : base(options) { }
}
```

URLs:
- `GET /api/previews/{key}` → raw bytes
- `GET /api/previews/{key}?w=300&fit=cover` → on-the-fly resize

### After (v0.9)

```csharp
[Route("api/previews")]
public sealed class PreviewsController : StorageMediaController<PreviewImage>
{
    public PreviewsController(IOptions<StorageMediaOptions> options) : base(options) { }
}
```

URLs:
- `GET /api/previews/{key}` → raw bytes (unchanged)
- `GET /media/{key}?w=300&fit=cover` → on-the-fly resize via the
  recipe pipeline's MediaController (different route)

**Migration steps:**

1. Replace `MediaContentController<T>` with `StorageMediaController<T>`.
2. Replace `IOptions<MediaContentOptions>` with `IOptions<StorageMediaOptions>`.
3. If the controller's `ConfigureResponse` override forced a content
   type (e.g. hardcoded `"image/jpeg"`), remove the hardcoding —
   the pipeline now stores the actual encoder output's content
   type, and `StorageMediaController` honours it via
   `ResolveContentType`.
4. Update SPA / client URLs that hit `?w=...&fit=...` to use
   `/media/{id}?...` instead.

## 4. Replace `IMediaTransformCache` with Standard HTTP Caching

v0.8's `IMediaTransformCache` was an opaque variant cache layered on
top of the controller. v0.9 relies on standard HTTP caching:

- **ETag** = `(sourceShortHash, recipeFingerprint)`.
  Deterministic; same source + same recipe = same ETag forever.
- **If-None-Match** → 304 short-circuit on the controller side.
- **Cache-Control** = `MediaWebOptions.DefaultCacheControl`
  (default `public, max-age=3600, stale-while-revalidate=86400`).

Browsers, CDNs, and reverse proxies handle the cache. There's no
application-side cache to register.

If you had a custom `IMediaTransformCache` implementation, drop it.
If you relied on the opaque cache for "regenerate everything",
trigger your variant pre-warmer (an admin endpoint that calls
`MaterializeAsync` for each known recipe).

## 5. Replace Implicit Operator Resolution with Recipes

v0.8 derived the transform from query params (`?w=300&fit=cover`
→ resize + fit operator). v0.9 expects a named recipe seed; bare
params build an ad-hoc recipe (allowed by default; disable in
production via `Koan:Media:Web:AllowAdHoc=false`).

### Before (v0.8)

Operators were registered via DI; the controller looked them up by
canonical param name. Ad-hoc URL → operator chain → output.

### After (v0.9)

Define recipes once, then either point clients at the named recipe
or let them build an ad-hoc one:

```csharp
[MediaRecipe("thumb-400",
    Description = "400x400 cover, WebP q70",
    Mutators = MutatorKind.Common)]
public static MediaRecipe Thumb400() => MediaRecipe.New()
    .ResizeCover(400, 400)
    .EncodeAs("webp", 70);
```

```
GET /media/{id}/thumb-400              # named recipe
GET /media/{id}/thumb-400?w=200        # named recipe with size override (Mutators allows it)
GET /media/{id}?w=400&h=400&fit=cover  # ad-hoc, no recipe seed
```

For ad-hoc to work in production, leave `AllowAdHoc=true`. For
strict production deployments, set it false and reject anything
without a named recipe.

## 8. Checklist

A short pass to confirm migration is complete:

- [ ] Every `using Koan.Media.Core.Extensions;` is replaced with `using Koan.Media.Core.Pipeline;`
- [ ] Every `.ResizeFit(...).ConvertFormat("jpeg", ...)` chain is replaced with `.AsMedia().ResizeFit(...).PreserveFormat(...)` or `.FlattenTo("jpeg", ...)` (explicit destruction)
- [ ] Every `MediaContentController<T>` is now `StorageMediaController<T>`
- [ ] Every `IOptions<MediaContentOptions>` is now `IOptions<StorageMediaOptions>`
- [ ] Any custom `IMediaTransformCache` registrations are removed
- [ ] Any `Koan:Media:Transforms:*` / `Koan:Media:Content:*` / `Koan:Media:Cache:*` config keys are removed or migrated
- [ ] Per-entity URL patterns that included transform query params now point at `/media/{id}?...` instead
- [ ] Variant pre-warmers (if any) use `MaterializeAsync` instead of looping through operator runs
- [ ] At least one `[MediaRecipe]` is defined for each visual variant the app exposes
- [ ] `IMediaSource` is registered so the `/media/{id}` controller can resolve source bytes
